What is Machine Learning?

Machine Learning is a way of teaching a computer to find patterns in data — without writing explicit rules. Instead of telling the program "if age > 40 AND education = Masters then income is high", you just show it thousands of examples and it figures out the patterns itself.

🗄️
Training Data
Historical examples with known answers — the model learns from these.
🧠
Model
The algorithm that finds patterns. In this project: SVM, Logistic Regression, Decision Tree.
🎯
Prediction
Given new, unseen data, the trained model predicts the answer.
📏
Evaluation
We measure how good the model is using metrics like Accuracy, Precision, Recall, F1.

// 02 — File 1

preprocessing.py

This is the first file that runs. Its job is to take the raw, messy CSV files and transform them into clean, numerical data that machine learning models can understand. It saves the result as new CSV files.

Why preprocessing? ML models only understand numbers. Text values like "Male", "Private", "Bachelors" must be converted. Also, large numbers like fnlwgt (~300,000) would dominate over small numbers like age (~40) — scaling fixes this.

→ clean(df)

preprocessing.py
python
def clean(df):
    df.columns = df.columns.str.strip()
    for col in df.select_dtypes(include="object").columns:
        df[col] = df[col].str.strip().str.rstrip(".")
    return df

What it does: removes extra spaces from column names and from string values, and removes trailing dots from values.

Why it matters: without this, "Male" and " Male" are seen as two different values. The encoder would fail on the test data because it learned "Male" but sees " Male".

→ detect_outliers(column, df)

preprocessing.py
python
def detect_outliers(column, df):
    Q1 = df[column].quantile(0.25)
    Q3 = df[column].quantile(0.75)
    IQR = Q3 - Q1
    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR
    df[column] = df[column].clip(lower=lower_bound, upper=upper_bound)

An outlier is an extreme value that's far from the rest — like someone with a capital-gain of $99,999 when most people have $0. These extreme values hurt the model because they distort patterns.

1
Calculate Q1 and Q3
Q1 = 25th percentile (bottom quarter), Q3 = 75th percentile (top quarter). IQR = Q3 − Q1 = the middle 50% of the data.
2
Define bounds
Lower bound = Q1 − 1.5×IQR. Upper bound = Q3 + 1.5×IQR. Anything outside these is an outlier.
3
Clip — not delete
clip() replaces outliers with the boundary value instead of deleting the whole row. This preserves the dataset size while removing the damaging effect of extreme values.

Applied to: age, fnlwgt, education-num, capital-gain, capital-loss, hours-per-week

→ Encoding — Converting Text to Numbers

Models cannot work with text. Two encoding strategies are used depending on the column type:

🏷️
Label Encoding
Used for binary columns — sex and Income. Maps two values to 0 and 1. Safe here because there's no false ordering between just two values.
🔢
One-Hot Encoding
Used for multi-class columns like workclass, education, occupation, etc. Creates a separate binary column for each category value.
preprocessing.py — Label Encoding
python
# Train
processed_data["sex"]    = labeldata("sex", processed_data)
processed_data["Income"] = labeldata("Income", processed_data)
# Result: Female=0, Male=1   |   <=50K=0, >50K=1

# Test — uses transform() only, NOT fit_transform()
processed_data_test["sex"]    = labeldatatest("sex", processed_data_test)
processed_data_test["Income"] = labeldatatest("Income", processed_data_test)
Critical rule: On test data, always use transform() — never fit_transform(). The test data must use the exact same mapping the model learned from training data. If we fit again on test data, the mappings might differ.

One-Hot Encoding example — the sex column was binary so Label Encoding worked. But for workclass with values like Private, Self-emp, Gov, etc.:

Original: workclass = "Private"
After OHE: workclass_Private=1, workclass_Self-emp=0, workclass_Gov=0, ...

This creates many new columns but avoids the model thinking "Self-emp (2) is twice Private (1)", which would be a false mathematical relationship.

→ MinMax Scaling

preprocessing.py
python
scale = MinMaxScaler()
# Train: fit AND transform
processed_data[["age", "fnlwgt", ...]] = scale.fit_transform(processed_data[[...]])
# Test: transform ONLY — uses min/max learned from train
processed_data_test[["age", "fnlwgt", ...]] = scale.transform(processed_data_test[[...]])
scaled_value = (x − min) / (max − min) → result is always between 0 and 1

Without scaling, fnlwgt (~300,000) would dominate age (~40) and education-num (~9) in the model's calculations, making those features effectively irrelevant.

The encoders, OneHotEncoder, and scaler are all saved with joblib so that the predictor can reuse them without rerunning preprocessing.

train_models.py — SVM

→ What is SVM?

The Support Vector Machine draws an optimal boundary between two classes. In 2D it's a line, in higher dimensions it's a hyperplane. The key idea is to maximize the Margin — the empty space between the boundary and the nearest data points.

Decision Boundary
The line (or hyperplane in high dimensions) that separates the two classes. Every prediction is made by checking which side of this boundary a new data point falls on.
Margin
The gap between the boundary and the nearest points from each class. SVM maximizes this gap — a wider margin means better generalization to unseen data.
Support Vectors
The data points sitting right on the edge of the margin. These are the only points that actually define the boundary — the rest don't matter.
RBF Kernel
In real data, classes aren't always linearly separable. The RBF (Radial Basis Function) kernel transforms the data so that non-linear patterns can be separated with a curved boundary.

→ Code Walkthrough

train_models.py
python
import pandas as pd
import pickle
from sklearn.svm import SVC
from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score, confusion_matrix

# Load already-processed data
train = pd.read_csv("Preprocessing/processed_train_data.csv")
test  = pd.read_csv("Preprocessing/processed_test_data.csv")

# X = features (107 columns), y = target (0 or 1)
X_train = train.drop(columns=["Income"])
y_train = train["Income"]
X_test  = test.drop(columns=["Income"])
y_test  = test["Income"]

# Build model with hyperparameters
svm = SVC(kernel='rbf', C=1)

# Train — finds the optimal decision boundary
svm.fit(X_train, y_train)

# Predict — classify each person in test set
svm_pred = svm.predict(X_test)

# Save the trained model
pickle.dump(svm, open('SVMmodel.pkl', 'wb'))

kernel='rbf' — Radial Basis Function. Allows curved decision boundaries for complex patterns. Other options: linear (straight line), poly (polynomial curves).

C=1 — The regularization hyperparameter. Higher C = stricter model that tries to classify every training point correctly but risks overfitting. Lower C = simpler model that accepts some errors but generalizes better.

fit() — The actual training. The model analyzes all 32K rows and 107 features to find the optimal hyperplane. Takes several minutes on this dataset.

predict() — Each test person is placed in 107-dimensional space. The model checks which side of the boundary they're on and returns 0 or 1.

Accuracy — what % of all predictions were correct. (TN+TP) / Total = (11542+2087) / 16276 = 83.7%

Precision — when the model said >50K, how often was it right? TP / (TP+FP) = 2087 / 2975 = 70.2%

Recall — of all people who actually earned >50K, how many did the model detect? TP / (TP+FN) = 2087 / 3846 = 54.3%. This is the weakest metric — the model misses nearly half of the high-income individuals.

F1-Score — harmonic mean of Precision and Recall. Uses harmonic (not regular) mean because if either value is very low, it should pull the score down significantly.

F1 = 2 × Precision × Recall / (P + R) = 2 × 0.702 × 0.543 / 1.245 = 61.2%

Logistic Regression

Despite its name, Logistic Regression is a classification model, not regression. It calculates the probability that a given input belongs to class 1 (>50K), then uses a threshold (usually 0.5) to decide.

logistic.py
python
model = LogisticRegression(C=1.0, max_iter=1000)
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

C=1.0 — same regularization concept as SVM. Higher C = fits training data more closely.

max_iter=1000 — the number of optimization steps the algorithm takes. Default (100) was not enough for this dataset to converge, so it's increased to 1000.

Logistic Regression is much faster than SVM and produces comparable accuracy (~83.8%) while being far more interpretable — you can directly see which features have the most influence via the model's coefficients.

Decision Tree

A Decision Tree is a model that makes predictions by asking a series of yes/no questions about the features, splitting the data at each step until it reaches a decision. It's the most intuitive model to explain — it literally looks like a flowchart.

This model uses GridSearchCV to automatically test many combinations of hyperparameters and find the best one. This is called hyperparameter tuning.

decision_tree.py — GridSearchCV
python
param_grid = {
    'max_depth': [6, 8, 10, 12],
    'min_samples_split': [5, 10, 15],
    'min_samples_leaf': [5, 10, 15],
    'criterion': ['gini', 'entropy']
}
grid = GridSearchCV(estimator=model, param_grid=param_grid,
                   cv=5, scoring="accuracy", n_jobs=-1)
grid.fit(x_train, y_train)
max_depth
How many questions (levels) the tree can ask. Deep trees memorize training data (overfitting). Shallow trees generalize better.
min_samples_split
Minimum number of samples needed in a node before it can split again. Higher values = simpler tree.
criterion: gini/entropy
The formula used to measure impurity when deciding where to split. Both usually give similar results.
cv=5
5-Fold Cross Validation — splits training data into 5 parts, trains on 4, validates on 1, repeats 5 times. More reliable than a single train/test split.
n_jobs=-1 means "use all available CPU cores" — this makes GridSearchCV run in parallel, testing multiple parameter combinations at the same time.

The Decision Tree also uses feature_importances_ to show which features most influenced its decisions — capital-gain, education-num, and age tend to be the top contributors.

After collecting the user's input, the predictor applies the exact same preprocessing steps as preprocessing.py:

1
Clean the input
Strip spaces and trailing dots from string values.
2
Label encode sex
Uses the saved encoders dict to apply the same Male=1/Female=0 mapping.
3
One-Hot encode categorical columns
Uses the saved onehotencoder. Produces the same 108-column format the model expects.
4
Scale numeric columns
Uses the saved MinMaxScaler — applies the same min/max from training data.
5
reindex — critical step
user_df.reindex(columns=features, fill_value=0) — ensures the columns are in the exact same order the model was trained on. Any missing column gets filled with 0. Without this, the model would get confused.
6
Predict and decode
The model outputs 0 or 1. inverse_transform() converts it back to the human-readable label: ≤50K or >50K.

// 08 — Final

Conclusion

This project builds a complete, end-to-end income classification system. Starting from raw census data, it applies a full preprocessing pipeline, trains three different ML models, evaluates them rigorously, and deploys them in a live prediction interface.

Key insight: The most important lesson from this project is that preprocessing and evaluation are as important as the model itself. A well-preprocessed dataset with proper encoding and scaling directly determines whether the model learns correct patterns — and thorough evaluation using Precision, Recall, and F1 reveals weaknesses that Accuracy alone would hide.
