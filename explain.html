ما هو Machine Learning؟

الـ Machine Learning هو طريقة تعليم الكمبيوتر يلاقي patterns في الداتا من غير ما تكتبله قواعد صريحة. بدل ما تقوله "لو العمر أكبر من 40 والتعليم عالي يبقى الدخل >50K" — بتديله آلاف الأمثلة ويتعلم لوحده.

🗄️
Training Data
أمثلة تاريخية بإجاباتها — الـ model بيتعلم منها.
🧠
Model
الخوارزمية اللي بتلاقي الـ patterns. في المشروع ده: SVM، Logistic Regression، Decision Tree.
🎯
Prediction
بعد التدريب، لما تديه data جديدة، بيتنبأ بالإجابة.
📏
Evaluation
بنقيس أداء الـ model بـ Accuracy و Precision و Recall و F1-Score.

→ clean(df)

preprocessing.py
python
def clean(df):
    df.columns = df.columns.str.strip()
    for col in df.select_dtypes(include="object").columns:
        df[col] = df[col].str.strip().str.rstrip(".")
    return df

بتعمل إيه: بتشيل الـ spaces الزيادة من أسماء الأعمدة ومن قيم الـ strings، وبتشيل النقط من آخر القيم.

ليه مهمة: من غيرها، "Male" و " Male" هيبقوا قيمتين مختلفتين في نظر الـ encoder ويفشل على الـ test data.

→ detect_outliers(column, df)

preprocessing.py
python
def detect_outliers(column, df):
    Q1 = df[column].quantile(0.25)
    Q3 = df[column].quantile(0.75)
    IQR = Q3 - Q1
    lower_bound = Q1 - 1.5 * IQR
    upper_bound = Q3 + 1.5 * IQR
    df[column] = df[column].clip(lower=lower_bound, upper=upper_bound)

الـ Outlier هو قيمة متطرفة بعيدة عن الباقين — زي حد عنده capital-gain بـ 99,999 وباقي الناس عندهم 0. القيم دي بتأثر بشكل كبير على الـ model.

1
حساب Q1 و Q3
Q1 = الـ 25th percentile (ربع الداتا السفلاني)، Q3 = الـ 75th percentile (ربع الداتا الفوقاني). الـ IQR = Q3 − Q1 = منتصف 50% من الداتا.
2
تحديد الـ bounds
Lower bound = Q1 − 1.5×IQR. Upper bound = Q3 + 1.5×IQR. أي قيمة برا الحدود دي هي outlier.
3
Clip — مش Delete
الـ clip() بتبدل الـ outliers بقيمة الحد بدل ما تحذف الصف كله. ده بيحافظ على حجم الداتا وبيشيل تأثير القيم المتطرفة.

متطبق على: age, fnlwgt, education-num, capital-gain, capital-loss, hours-per-week

// 03 — ملف 2

SVM — ما هو وبيشتغل إزاي؟

الـ Support Vector Machine هو خوارزمية بترسم خط فاصل (Decision Boundary) بين الـ classes بأكبر مسافة ممكنة. المسافة دي اسمها Margin. في الداتا بتاعتنا (107 dimensions) الخط ده بيبقى hyperplane مش خط عادي.

Decision Boundary
الخط (أو الـ hyperplane في أبعاد أعلى) اللي بيفصل الـ classes. كل prediction بتتحدد من أي جانب من الخط ده الشخص.
Margin
المسافة بين الـ boundary والأقرب نقطة من كل class. الـ SVM بيحاول يكبّر الـ margin ده عشان الـ model يبقى أفضل على داتا جديدة.
Support Vectors
النقط اللي على حافة الـ Margin. دول بس اللي بيحددوا الخط الفاصل — باقي النقط مش مهمة.
RBF Kernel
في الداتا الحقيقية الـ classes مش دايمًا منفصلة بخط مستقيم. الـ RBF بيحول الداتا عشان يقدر يفصل بـ حدود منحنية.

→ الكود خطوة بخطوة

train_models.py
python
import pandas as pd
import pickle
from sklearn.svm import SVC
from sklearn.metrics import accuracy_score, precision_score,
                             recall_score, f1_score, confusion_matrix

# تحميل الداتا اللي اتعملها preprocessing
train = pd.read_csv("Preprocessing/processed_train_data.csv")
test  = pd.read_csv("Preprocessing/processed_test_data.csv")

# X = الـ features (107 عمود)، y = الـ target (0 أو 1)
X_train = train.drop(columns=["Income"])
y_train = train["Income"]
X_test  = test.drop(columns=["Income"])
y_test  = test["Income"]

# بناء الـ model بالـ hyperparameters
svm = SVC(kernel='rbf', C=1)

# التدريب — بيلاقي الـ decision boundary الأمثل
svm.fit(X_train, y_train)

# التنبؤ — بيصنف كل شخص في الـ test set
svm_pred = svm.predict(X_test)

# حفظ الـ model
pickle.dump(svm, open('SVMmodel.pkl', 'wb'))

X و y: الـ X هي الـ features — كل الأعمدة ما عدا Income. الـ y هي الـ target — عمود Income بس، فيه 0 (≤50K) أو 1 (>50K). ده الـ encoding اللي عمله preprocessing.py.

kernel='rbf': بيحدد شكل الـ decision boundary. الـ RBF بيخليه منحني مش مستقيم — أحسن للداتا المعقدة.

C=1: الـ regularization hyperparameter. C عالي = الـ model بيحاول يصنف كل نقطة تدريب صح بس ممكن يعمل overfitting. C منخفض = أبسط لكن بيقبل بعض الأخطاء.

fit(): التدريب الفعلي. الـ model بيحلل الـ 32K صف و107 عمود يلاقي الـ hyperplane الأمثل. بياخد دقايق.

predict(): كل شخص في الـ test بيتحط في الـ 107-dimensional space. الـ model بيشوف هو على أنهي جانب من الـ boundary ويرجع 0 أو 1.

pickle.dump(): بيحفظ الـ model في ملف .pkl عشان نستخدمه بعدين من غير ما نعيد التدريب.

Accuracy — 83.7%: كام بالمية من كل الـ predictions كانت صح. بتحسب (TN+TP) / الكل. يبدو كويس بس ممكن يكدب لو في class imbalance.

F1-Score — 61.2%: الـ Harmonic Mean بين الـ Precision والـ Recall. بيستخدم harmonic مش متوسط عادي عشان لو أي واحد منهم وطي جداً يأثر بشكل كبير.

F1 = 2 × Precision × Recall / (P + R) = 2 × 0.702 × 0.543 / 1.245 = 61.2%

Logistic Regression

على الرغم من اسمه، الـ Logistic Regression هو classification model مش regression. بيحسب الاحتمالية إن الشخص ده في class 1 (>50K)، وبعدين بيستخدم threshold (عادة 0.5) عشان يقرر.

logistic.py

model = LogisticRegression(C=1.0, max_iter=1000)

الـ Logistic Regression أسرع بكتير من الـ SVM وبيطلع accuracy قريب (~83.8%) مع ميزة إضافية: تقدر تشوف مباشرة أي features ليها أكبر تأثير من خلال الـ coefficients.

Decision Tree

الـ Decision Tree بيعمل predictions بإنه بيسأل سلسلة من الأسئلة الـ Yes/No عن الـ features، بيعمل split في كل خطوة لحد ما يوصل لقرار. ده أسهل model تفسره — حرفياً زي الـ flowchart.

الـ model ده بيستخدم GridSearchCV عشان يجرب كتير من combinations من الـ hyperparameters ويلاقي الأحسن أوتوماتيك. ده بيتسمى Hyperparameter Tuning.

decision_tree.py — GridSearchCV
python
param_grid = {
    'max_depth':        [6, 8, 10, 12],
    'min_samples_split': [5, 10, 15],
    'min_samples_leaf':  [5, 10, 15],
    'criterion':        ['gini', 'entropy']
}
grid = GridSearchCV(estimator=model, param_grid=param_grid,
                   cv=5, scoring="accuracy", n_jobs=-1)
grid.fit(x_train, y_train)
max_depth
عدد مستويات الأسئلة في الـ tree. tree عميقة = بتحفظ الـ training data (overfitting). شجرة ضحلة = بتـgeneralize أحسن.
min_samples_split
أقل عدد samples في node عشان تقدر تعمل split. قيمة عالية = tree أبسط.
criterion: gini/entropy
المعادلة المستخدمة لقياس الـ impurity عند قرار الـ split. الاتنين بيطلعوا نتايج قريبة عادة.
cv=5
5-Fold Cross Validation — بيقسم الـ training data لـ 5 أجزاء، بيتدرب على 4 ويتحقق على 1، ويكرر 5 مرات. أموثوق من split واحدة.
n_jobs=-1 معناها "استخدم كل الـ CPU cores المتاحة" — بيخلي الـ GridSearchCV يشتغل بشكل parallel ويجرب combinations كتير في نفس الوقت.
